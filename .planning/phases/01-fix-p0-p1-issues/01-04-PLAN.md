<objective>
Prevent accidental repo-wide pytest collection from the monorepo root by adding `pytest.ini` with `testpaths = ` (empty, so bare `pytest` from root collects nothing) and update the Makefile `test` target to call `scripts/run_portfolio_pytest.sh` instead of bare `pytest -v`.

This also serves as the verification checkpoint for P1-01 (portfolio.yaml already complete with 37 entries) and P1-02 (README.md badges already use tazzledazzle) — these are documented as already-resolved in this plan's summary.

Purpose: Running `pytest` from the monorepo root currently produces 64 collection errors due to ImportPathMismatchError and missing packages across different subproject trees. The CI already calls `scripts/run_portfolio_pytest.sh` (the safe per-suite runner), but the Makefile `test` target still runs `pytest -v --tb=short --continue-on-collection-errors` from root. Adding `pytest.ini` with `testpaths =` (empty) means accidental bare `pytest` calls from root collect nothing rather than triggering errors.
Output: `python3 -m pytest --collect-only -q` from repo root collects 0 items with 0 errors. `make test` calls `scripts/run_portfolio_pytest.sh`.
</objective>

    Do NOT set `--ignore` glob patterns — they are fragile across pytest versions.

    `python3 -m pytest --collect-only -q`
