<objective>
Add a per-package mypy step to the CI python job that avoids the duplicate `conftest` module name failure that occurs when running `mypy .` from the repo root.

The root CI python job (`ci.yml`) already uses `scripts/run_portfolio_pytest.sh` for testing but has no mypy step. The CONCERNS.md and P1-04 require that mypy be added in a form that does not fail on duplicate conftest module names across `ai-best-practices-examples/*/tests/conftest.py`.

The fix: run mypy with `--explicit-package-bases` flag OR scope mypy to specific subproject `src/` directories only (excluding `tests/`). Using `--ignore-missing-imports` is also needed since many subprojects lack type stubs.

Purpose: P1-04 requires "Root CI mypy does not fail on duplicate conftest module names." Since mypy was removed from CI previously, we add it back in a monorepo-safe form.
Output: CI python job has a mypy step using `--explicit-package-bases --ignore-missing-imports` that exits 0 on the current codebase.
</objective>
