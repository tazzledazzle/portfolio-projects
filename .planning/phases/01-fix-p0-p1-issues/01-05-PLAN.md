<objective>
Add a per-subproject mypy step to the CI python job that avoids the duplicate module name failures that occur when running `mypy .` from the repo root.

Two collision classes exist in this repo:
1. `ai-best-practices-examples/*/tests/conftest.py` — duplicate `conftest` module names (was the known issue).
2. `ci-cd-pipelines/*/src/main.py` — six subprojects each have a file named `main` under `src/`; `--explicit-package-bases` does NOT suppress this class of collision because each `src/main.py` resolves to the module name `main` regardless of the base directory.

The fix: create `scripts/run_portfolio_mypy.sh` (following the `run_portfolio_pytest.sh` pattern) that runs mypy per-subproject against specific `src/` directories, targeting only the subprojects that do not have module-name collisions. The CI python job calls this script instead of invoking `mypy .` directly.

Purpose: P1-04 requires "Root CI mypy does not fail on duplicate module names." Scoping mypy to collision-free subprojects achieves this while still providing meaningful type coverage.
Output: `scripts/run_portfolio_mypy.sh` (new file) and updated `.github/workflows/ci.yml`.
</objective>
