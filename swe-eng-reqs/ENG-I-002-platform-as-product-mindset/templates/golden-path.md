# Golden path — platform onboarding

This template describes the preferred self-service path for internal teams.

## Steps

1. Create a project in the IDP catalog (catalog owned by ENG-E-005).
2. Attach a CI pipeline and staging environment via self-service APIs.
3. Adopt the published SLA targets and track team adoption metrics here.
4. Prefer automation over tickets for routine environment requests.

## Why golden

A golden path reduces custom snowflake setups, shortens time-to-first-deploy, and
makes platform success measurable through adoption and SLA adherence.
