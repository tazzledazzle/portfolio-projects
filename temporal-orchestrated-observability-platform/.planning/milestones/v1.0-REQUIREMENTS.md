# Requirements: Temporal-Orchestrated Observability Platform

**Defined:** 2026-05-19  
**Core Value:** An operator can start an AI workflow, watch it in Temporal, and in Grafana trace the same run across metrics, traces, and logs.

## v1 Requirements

### Foundation

- [x] **FOUND-01**: `docker compose up` brings Temporal, Grafana, Prometheus, Loki, Tempo (and documented profiles) to healthy state.
- [x] **FOUND-02**: `./scripts/smoke.sh` exits 0 on clean checkout (waits for health endpoints).
- [x] **FOUND-03**: CI runs Gradle build + unit tests for worker module on push to `main`.

### Temporal orchestration

- [x] **TEMP-01**: Kotlin worker connects to Temporal and polls task queue `ai-workflows`.
- [x] **TEMP-02**: Temporal UI reachable; workflow history visible for sample runs.
- [x] **TEMP-03**: Worker graceful shutdown drains polls without corrupting tasks (documented behavior).

### AI workflows

- [x] **WF-01**: **RAG Q&A** workflow completes with stubbed retrieval + LLM activities.
- [x] **WF-02**: **Agent tools** workflow runs plan → tool → synthesize sequence with retries configured.
- [x] **WF-03**: **Batch eval** workflow fans out N scoring activities and aggregates results.
- [x] **WF-04**: CLI or HTTP starter can trigger each workflow with synthetic input.

### Instrumentation (Kotlin + OTel)

- [x] **OTEL-01**: Every activity execution creates an OTel span with `workflow_type`, `workflow_id`, `run_id`, `activity_type`.
- [x] **OTEL-02**: Prometheus endpoint exposes `activity.duration` histogram and `workflow.completed` counter with bounded labels.
- [x] **OTEL-03**: Structured JSON logs include `trace_id` and Temporal identifiers when span context is active.
- [x] **OTEL-04**: Outbound HTTP (LLM stub) creates child spans with W3C trace context propagation.

### Trace and metrics backends

- [x] **BACK-01**: OTLP traces visible in **Jaeger** UI for a sample workflow run.
- [x] **BACK-02**: Prometheus scrapes worker metrics; targets show `UP` in Prometheus UI.
- [x] **BACK-03**: OTel Collector config validates (`otelcol validate` or equivalent) in CI.

### LGTM / Grafana

- [x] **LGTM-01**: Loki ingests worker logs; Grafana Explore returns lines by `workflow_id`.
- [x] **LGTM-02**: Tempo receives traces; Grafana Explore shows trace by ID from log derived field.
- [x] **LGTM-03**: Provisioned dashboards: Workflow Overview, Activity Latency, LLM Proxy (stub metrics OK).
- [x] **LGTM-04**: Dashboard links documented in `docs/OPERATIONS.md`.

### Operations

- [x] **OPS-01**: `docs/OPERATIONS.md` covers stuck workflow, missing traces, Prometheus target down, Loki disk pressure.
- [x] **OPS-02**: Example alert rules (Prometheus YAML) for high activity failure rate and export errors.
- [x] **OPS-03**: Manual validation record for each runbook path in phase verification doc.

## v2 Requirements

- **K8S-01**: Helm chart or Kustomize for Temporal + observability stack
- **K8S-02**: Temporal mTLS between worker and server
- **AI-01**: Real OpenAI/Anthropic integration behind feature flag
- **AI-02**: LangGraph activity embedding inside Temporal activities

## Out of Scope

| Feature | Reason |
|---------|--------|

| Custom embedding training | Not observability focus |
| SaaS auth/billing | Demo platform |
