# Security Policy

## Supported Versions

Security fixes are applied to the `main` branch of this portfolio repository. Individual demo subprojects may not receive long-term support beyond what is needed to keep the portfolio buildable and documented.

## Reporting a Vulnerability

If you discover a security issue in this repository:

1. **Do not** open a public GitHub issue for sensitive reports.
2. Email the repository owner via the contact address on the [GitHub profile](https://github.com/tazzledazzle).
3. Include reproduction steps, affected paths, and potential impact.

## Response Expectations

- Acknowledgement within 5 business days
- Triage and severity assessment within 10 business days
- Fix or mitigation guidance for confirmed issues affecting runnable demo code

## Scope Notes

Many projects in this monorepo are educational demos with intentionally simplified authentication and mock credentials. Reports about demo-only weaknesses (for example, fake password hashing in `ws-chat-fast`) are welcome but may be addressed by documentation rather than production-grade hardening.
